ㅇ 메서드(Method) : 프로그램에서 특정 작업을 수행하기 위한 코드의 집합.
 - 함수(Function)과 유사한 개념
ㅇ 메서드의 필요성 : 같은 코드를 *반복해서 작성하는 작업*을 효율적으로 수행.
ㅇ 메서드의 장점
 - 소스 코드의 재사용성 : 중복 코드 작성을 방지하고, 재사용성을 향상.
 - 소스 코드의 가독성 : 비슷한 성격의 관련 작업을 하나로 모아 가독성을 향상.
 - 소스 코드의 유지보수성 : 유지 및 리펙터링 용이
 
 ㅇ 메서드의 유형 : 표준 라이브러리 & 사용자 정의
  - 표준 라이버러리 메서드(사전 정의 메서드 & 내장 메서드) : 자바 클래스 라이브러리에 
  이미 정의된 메서드로 호출 즉시 사용 가능.
  ex) System.out 클래스에 println() 메서드
  
  - 사용자 정의 메서드 : 사용자 또는 프로그래머가 작성한 메서드
  ex) add() 메서드 
  
 ㅇ 사용자 정의 메서드의 종류
  - 정적 메서드(static method) : static 키워드를 선언하는 메서드. 호출시 클래스 객체 불필요.
  - 인스턴스 메서드(instance method) : non-static 클래스에 속하는 메서드. 
  호출시 클래스 객체 또는 클래스 인스턴스 필요.
  
 ㅇ 메서드의 기본 구조
 
  public 	static void 	main 	(String[] args) 	=> 선언부
  접근제한자			반환유형 메서드명	매개변수 목록 			=> 구현부    
  
 ㅇ 접근제한자(access modifier) : 선택 사항. 메서드의 접근 유형 정의.
 ㅇ 반환 유형(return type) : 메서드가 반환(return)하는 유형. 
 반환 값이 없는 경우 void로 작성.
 ㅇ 메서드명(method name) : 메서드가 호출될 때 사용되는 이름. 
  - 메서드명은 *소문자로 시작*.
  - 메서드명은 *동사*형태.
  - 메서드명에 2개 이상의 단어가 포함되는 경우 동사로 시작하고, 그 뒤에는 형용사 또는
  명사가 와야함.
  (ex: add(), getName())
  - 메서드명에 3개 이상의 단어가 포함되는 경우 첫 번째 단어를 제외하고 나머지 단어의
  첫 글자는 대문자로 시작.
  (ex: getAddressDetails())
 ㅇ 매개변수 목록(parameter list) : 유형, 순서 개수로 구성. 매개변수는 선택 사항.
 ㅇ 메서드 본문 : 메서드의 코드를 수행하는 작업을 정의. 중괄호로 본문을 묶음. 호출 시
 중괄호 안의 코드가 실행되어 값을 반환. void의 경우 메서드값을 반환하지 않음.
