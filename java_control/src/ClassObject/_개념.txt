Chapter 7. 클래스와 객체

7.1 객체지향 프로그래밍
1. 개념: 데이터와 메서드를 포함하는 '객체' 개념에 기반한 프로그래밍
		프로그래밍의 유연성을 향상, 유지 & 관리의 가능성을 높임.
		데이터 및 데이터의 동작(메서드)을 단일 위치(객체)에 통합하여 프로그램 작동 방식을 더 쉽게 이해.
		
	ㅇ 장점
	 - 개발 속도 향상: 더 빠르고 쉽게 실행
	 - 소프트웨어 유지&관리 향상: 프로그램에 대한 명확한 구조 제공
	 - 소프트웨어 개발의 생산성 향상: 코드 반복이 줄어들고, 코드의 재사용, 수정, 디버깅이 용이
	 - 개발 비용 절감: 더 적은 코드, 더 짧은 개발 시간으로 완전히 재사용 가능한 애플리케이션 제작 가능
	 
	ㅇ 객체지향 프로그래밍의 구성요소
	 - 클래스(class): 같은 종류의 집단에 속한 속성과 행동을 정의한 틀.
	 - 객체(object): 클래스의 인스턴스. (클래스라는 설계도로 만들어진 물건, 실체)
	 - 캡슐화(encapsulation): 데이터와 행동을 하나의 단위로 묶는 기법. - 일명 접근제한자.
	 - 상속(inheritance): 이미 존재하는 한 클래스의 멤버(변수, 메서드)를 다른 클래스에 물려주는 기법.
	 - 다형성(polymorphism): 변수, 메서드 또는 객체가 여러 형태를 취하는 기법.
	 - 추상화(abstraction): 불필요한 내부 세부 사항을 숨기고 필수 사항을 표시하는 것.

2. 캡슐화: 데이터(변수)와 행동(메서드)을 하나의 단위(클래스)로 묶는 기법.
		 데이터를 수정하지 못하도록 안전하게 하기 위해 정보의 세부 사항을 숨기고 객체의 데이터와 메서드를 보호하는 과정.
		 private 접근제한자로 클래스의 변수 선언.
		 클래스의 변수에 접근할 수 있는 public 접근제한자로 선언한 Getter(), Setter() 메서드를 제공
		 Getter()는 변숫값을 가져올 수 있는 메서드이고, Setter()는 변수에 값을 저장할 수 있는 메서드.
		 
3. 상속: 이미 존재하는 한 클래스의 멤버(변수, 메서드)를 다른 클래스에 물려주는 기법.
		코드를 재사용하고 두 클래스 간 부모-자식 관계를 설정.
		
4. 다형성: 여러 형태를 취한 다는 의미로, 객체지향 프로그래밍에서는 변수, 메서드 ,객체가 여러 형태를 취하는 기법을 의미.
		 상속에 의해 서로 관련된 하나 이상의 클래스 또는 객체가 있을 때 발생.
		 메서드 오버로딩(method overloading)과 메서드 오버라이딩(method overriding)의 개념 존재.
	ㅇ 메서드 오버로딩(method overloading)
	 - 메서드명이 같지만 메서드에 전달된 매개변수의 자료형 또는 개수가 다른 둘 이상의 메서드를 갖는 것.
	 
	ㅇ 메서드 오버라이딩(method overriding)
	 - 상위 클래스의 메서드를 하위 클래스에서 재정의하는 것.
	 
5. 추상화: 불필요한 세부 정보를 포함하지 않고 객체의 공통적인 속성과 행동만을 나타내는 것.
		 불필요한 세부 정보를 숨겨서 프로그래밍의 복잡성과 노력을 줄이는 것이 주 목적.
		 추상 클래스(abstract class)와 인터페이스(interface)를 이용하여 구현
		 
	ㅇ 추상 클래스(abstract class)
	
	ㅇ 인터페이스(interface)

7.2 클래스와 객체
1. 클래스와 객체의 관계
	ㅇ 객체지향 프로그래밍: 객체를 조립하여 프로그램을 만드는 것.
	
2. 클래스의 설계 및 구현:
	ㅇ 클래스: 특정 유형의 객체를 만들기 위한 설계도 또는 탬플릿.
			 객체가 어떻게 작동하고 객체가 무엇을 할 것인지 결정.
			 객체에 어떤 속성(상태)이 있고, 어떤 기능(행동)을 하는지 설명하는 설계도.
	ㅇ 클래스 설계 - 클래스 구현
	
3. 객체: 클래스의 인스턴스. 클래스 자체의 복사본. 특정 유형의 데이터를 유용하게 만드는 메서드와 변수(속성)로 구성.

7.3 클래스의 구성 요소
1. 멤버 변수(member variable): 클래스의 속성. 객체의 데이터, 메서드의 반환값, 상태 등을 저장하는 곳.

2. 멤버 메서드(member method): 객체의 동작, 데이터의 조작이 이루어지는 곳.

7.4 생성자: 객체 생성시 new 연산자를 사용하여 호출되는 부분.
		  클래스의 객체가 생성될 때마다 자동으로 호출되고, 객체를 초기화하는 데 이용.
		  생성자명은 클래스명과 같고 반환 유형이 없음.
	- 생성자명은 해당 클래스명과 동일해야 한다.
	- 생성자는 abstract, final, static을 선언할 수 없다.
	- 접근제한자는 생성자의 접근을 제어하는 데 사용한다. 즉, 다른 클래스가 생성자를 호출할 수 있다.
	
	매개변수가 없는 생성자 & 매개변수가 있는 생성자
	
1. 기본 생성자(default constructor): 매개 변수가 없는 생성자. 클래스명();
