8.1 상속
1. 상속의 필요성: 기존 클래스의 기능을 사용하여 새 클래스를 만드는 기술.
한 클래스가 다른 클래스의 특징(멤버 메서드와 변수)을 가져오도록 하는
자바 객체지향 프로그래밍의 필수적인 부분.

2. 상속의 개념

3. Is-A 관계

8.2 상속의 유형
1. 단일 상속: 클래스가 하나의 클래스에 의해서만 확장되는 것.
부모 클래스와 자식 클래스가 각각 하나뿐인 경우.

2. 다단계 상속: 클래스가 하나의 클래스에 상속하고, 상속받은 자식 클래스가 또 다른
클래스에 상속하는 것.

3. 계층적 상속: 단일 부모 클래스의 값과 메서드를 여러 자식 클래스에 전달하는 것.

4. super를 이용한 부모 클래스 참조
 -super: 부모 클래스의 멤버 요소인 변수와 메서드에 접근할 수 있는 명령어.
 자식 클래스의 메서드에서 호출하여 부모 클래스의 변수나 메서드에 접근.

 8.3 다형성
 1. 다형성의 필요성
  - 하나의 객체가 여러 가지 유형을 띠는 특성.
  - 다형성을 통해 상속과 메서드 재정의를 활용하여 확장성 있는 프로그램 생성 가능.
  - 상위 클래스에서 공통적인 부분을 제공하고, 하위 클래스에서 각 클래스에 맞는 기능 구현.
  
  ㅇ 다형성 활용의 이점
   - 단일 메서드를 다른 클래스에서 다르게 작동 가능
   - 동일한 구현에 대해 다른 메서드명을 선언하지 않아도 됨
   - 상속을 더 쉽게 구현 가능
   
 2. 다형성의 개념
  - 상속에 의해 서로 관련된 하나 이상의 클래스 또는 객체가 존재할 때 발생.
  - 다형성을 띈 메서드를 사용하여 다른 작업 수행 가능.
  
 8.4 다형성의 유형
 1. 컴파일타임 다형성(compile-time polymorphism)
  - 정적 다형성
  - 연산자 오버로딩과 메서드 오버로딩으로 구현.
  - 메서드 오버로딩: 메서드명이 같지만 다른 매개변수를 사용하는 프로세스.
  
 2. 런타임 다형성: 동적 메서드로 재정의된 메서드에 대한 메서드 호출이 런타임에 해결되는
 프로세스. 메서드 오버라이딩을 통해 구현
  - 메서드 오버라이딩: 부모 클래스로부터 상속받은 메서드를 자식 클래스에서 특정한 형태로
  재정의하는 것. 메서드 재정의라고도 불림.
