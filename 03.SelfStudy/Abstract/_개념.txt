Chapter 09. 추상 클래스와 인터페이스

9.1 추상화
1. 추상화의 개념: 불필요한 정보를 숨기고 중요한 정보만을 나타내는 것.
어떤 영역에서 필요한 공통의 속성이나 행동을 추출하여 효율적 코드 작성을 도모.
 ㅇ 장점
  - 객체 간 복잡성의 감소
  - 코드의 중복을 막고 재사용성을 높임.
  - 사용자에게 중요한 세부 정보만 제공함으로써 응용 프로그램이나 프로그램 보안에 도움.
  
2. 추상화의 유형
ㅇ 데이터 추상화: 복잡한 자료형을 만들고 구현을 숨기는 것.
 - 연결 목록(Linked list), 배열 목록(Array list), 해시맵(HashMap)
 - 사용자에게 노출되는 동작을 변경하지 않고 얼마든지 구현을 변경 가능

ㅇ 제어 추상화
 - 작업의 단위 정의를 만들고 필요할 때마다 재사용하는 것.
 - 반복되는 모든 코드를 수집하고 이를 하나의 단위로 노출.
 
3. 추상화 구현 방법: 추상 클래스 또는 인터페이스를 통해 구현.
 ㅇ 추상 클래스
  - 한 번에 하나의 클래스 또는 추상 클래스만 상속
  - 다른 구체적인(일반) 클래스 또는 추상 클래스를 상속
  - 추상 메서드와 구체적인 메서드를 모두 가짐
  - 메서드를 추상으로 선언하는 데 abstract 키워드가 필요
  - 접근제한자 protected, public이 선언된 추상 메서드를 가짐
  - 모든 접근제한자와 함께 static, final, static final 변수를 가짐
  
 ㅇ 인터페이스
  - 한 번에 원하는 만큼만 인터페이스를 상속할 수 있음
  - 다른 인터페이스만 상속 가능
  - 추상 메서드만 가질 수 있음
  - 메서드를 추상으로 선언하는 데 abstract 키워드는 선택 사항
  - 접근제한자 public이 선언된 추상 메서드를 가질 수 있음
  - static final(상수) 변수만 가질 수 있음
  
9.2 추상 클래스: 이론적인 정의 역할만 할 수 있고 객체를 만들 수 없으나, 추상 클래스가 아닌 자식 클래스를 만들 수 있음
1. 추상 클래스의 선언: abstract 키워드를 사용. 일반 메서드 뿐 아니라 추상 메서드도 포함할 수 있음.
 ㅇ 추상 메서드
  - 추상 메서드는 자식 클래스에서 구현
  - 추상 메서드를 포함할 수도 , 포함하지 않을 수도 있음
  - 클래스에 추상 메서드가 포함될 시 반드시 추상 클래스로 선언해야
  - 추상 클래스는 이론적인 정의 역할만 가능하고, 객체를 만들 수는 없음

 ㅇ 추상 클래스 사용 시 주의 사항
  - 항상 abstract 키워드로 선언해야
  - 추상 클래스 내에도 일반적인 메서드가 포함될 수 있음
  - 클래스의 메서드 중 하나가 추상 메서드이면 해당 클래스는 추상 클래스여야
  - 추상 클래스를 선언 후 자식 클래스에서 상속받을 경우 해당 메서드를 재정의해야
  - 추상 클래스는 자신의 인스턴스를 가질 수 없음
  - 프로그램이 자식 클래스의 객체를 만들 때, 컴파일러가 추상 클래스의 생성자를 호출
  - 상속만 가능하고 객체를 생성할 수 없는 클래스를 생성하려면 추상 클래스 내부에 일반 메서드만 가질 수 있음
  - 단일 클래스가 여러 추상 클래스를 상속받는 건 불가능. 다중 상속은 인터페이스에서 사용.
  - 추상 클래스에 final() 메서드를 포함할 수 있음. 단, final 클래스는 추상 메서드를 가질 수 없음

2. 추상 클래스의 상속: 자체의 인스턴스를 만들 수 없으므로 상속해서 사용. 자식 클래스는  추상 클래스를 상속 받고 추상 메서드를 재정의하여 사용
