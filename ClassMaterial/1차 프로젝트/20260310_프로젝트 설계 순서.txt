요구사항 작성 팁

요구사항은 사용자가 할 수 있는 행동 중심으로 작성한다. 
 - 사용자는 회원 가입을 한다. 일기를 작성한다. 댓글을 작성한다.

 - 게시판 기능, 로그인 시스템, 사용자 관리 -> 이렇게 쓰지 마시오!!

한 요구사항에는 하나의 기능만 작성한다.
요구 사항은 구체적으로 작성한다. 
 - 게시글을 관리할 수 있다. -> 이렇게 쓰지 마시오!!

 - 게시글을 작성할 수 있다., 게시글을 수정할 수 있다., 게시글을 삭제할 수 있다.
위와 같이 기능을 세분화할 것!!

화면 기준으로 생각하면 작성하기 쉽다!!
대상 별로 나누어서 작성한다.(사용자, 관리자 등)

<요구사항 작성 요령>
1. ID 규칙을 만들어라
2. 요구사항 내용은 문장형으로 작성한다. (누가 + 무엇을 한다)
3. 우선순위를 지정한다. 
     높은 - 반드시 구현, 중 - 구현 권장, 낮음 - 추가 기능
4. 테스트 시나리오를 같이 작성한다. 
     로그인 페이지 접속 -> 이메일 입력 -> 비밀번호 입력 -> 로그인 버튼 클릭 -> 로그인 성공 확인

<요구사항 작성 시 주의 사항>
1. 너무 추상적으로 작성하지 않는다. 
2. 기술 중심으로 작성하지 않는다. 
3. 너무 큰 기능을 하나의 요구 사항에 작성하지 않는다. 
4. 개발 방법을 작성하지 않는다. 

<기능 목록 정의서 작성>
- 어떤 기능을 제공하는지 정리한 문서
1. 기능은 사용자가 할 수 있는 행동으로 작성한다. 
2. 기능을 최대한 작게 나눈다. 
3. 사용자 기준으로 작성한다.

<입력데이터와 출력데이터 작성 방법>
입력데이터 : 기능을 수행하기 위해 들어오는 값
출력데이터 : 기능 수행 결과로 나오는 값

입력데이터는 사용자가 작성하거나, 서버가 받아야 하는 값
출력데이터는 처리 후 화면에 보여주거나, 저장 결과(DB)

입력데이터는 
1. 사용자가 입력하는 값 : 이메일, 비밀번호, 제목, 댓글 등
2. 시스템이 넘겨주는 값 : 로그인한 사용자ID, 게시글 ID, 날짜 등
3. 다른 화면이나 기능에서 전달되는 값 : 선택한 게시글 ID, 수정한 일기 ID

출력데이터는
1. 화면에 표시되는 결과: 게시글 목록, 로그인 성공메시지, 댓글 목록, 그래프
2. 시스템 처리 결과 : 저장 성공, 수정완료, 삭제 완료, 실패 메시지
3. 다음 단계로 넘겨주는 값 : 회원 정보 객체, 일기 상세 데이터, 공감수 변경 결과

입력데이터 - 이 기능을 실행하려면 무엇이 필요한가?
출력데이터 - 이 기능이 끝나면 무엇이 나오는가?
